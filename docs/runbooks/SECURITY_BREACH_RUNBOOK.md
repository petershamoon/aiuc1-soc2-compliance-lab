# AI Failure Runbook: Security Breach

**Control**: AIUC-1 E001

**Purpose**: This runbook provides the immediate steps to take in response to a suspected security breach involving any of the AI agents or the project's Azure infrastructure.

## 1. Triage & Identification

**Objective**: Confirm the breach and identify the affected components.

| Step | Action | Owner | Evidence |
| :--- | :--- | :--- | :--- |
| 1.1 | **Receive Alert**: Receive a security alert from Microsoft Defender for Cloud, Azure Monitor, or manual observation. | Pete | Alert Screenshot |
| 1.2 | **Isolate Agent**: Immediately disable the suspected agent's access in Azure AI Foundry to prevent further action. | Pete | Azure Portal Screenshot |
| 1.3 | **Identify Compromise**: Review agent logs in Application Insights and Azure resource logs to determine the nature and scope of the breach (e.g., unauthorized access, data exfiltration, malicious tool use). | Pete | App Insights Query Results |

## 2. Containment

**Objective**: Limit the impact of the breach.

| Step | Action | Owner | Evidence |
| :--- | :--- | :--- | :--- |
| 2.1 | **Rotate Credentials**: Rotate all secrets and keys associated with the compromised components (e.g., Service Principal secret, OpenAI API key, SQL password). | Pete | Key Vault Secret Version History |
| 2.2 | **Review IAM**: Audit all IAM and RBAC assignments on the subscription, resource groups, and Azure AI Foundry project. Remove any unauthorized or overly permissive roles. | Pete | `az role assignment list` Output |
| 2.3 | **Snapshot Resources**: If data corruption is suspected, take snapshots of affected disks and databases. | Pete | Azure Portal Screenshot |

| Step | Action | Owner | Evidence |
