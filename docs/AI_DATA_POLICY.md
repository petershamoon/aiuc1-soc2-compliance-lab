# AI Data Policy

**Controls:** AIUC-1 A001 (Input Data Policy), A002 (Output Data Policy)

**Purpose:** This document establishes data handling policies for the SOC 2 Learning Agent.

## Input Data Policy (A001)

### What Data the Agent Processes

The SOC 2 Learning Agent processes the following input data:

- **Azure resource configurations** retrieved via Azure Management APIs (storage accounts, NSGs, SQL servers, RBAC assignments, policy states, Defender scores)
- **User prompts** submitted through the Azure AI Foundry Playground
- **Tool parameters** passed to the 12 Azure Functions

### Model Training

- The agent uses OpenAI's `gpt-4.1-mini` model via Azure AI Foundry
- Per OpenAI's API data usage policies, data submitted via the API is **not** used for model training
- No fine-tuning or custom training is performed on any model

### Data Retention

- User prompts and agent responses are retained within the Azure AI Foundry conversation history for the duration of the project
- Azure resource configuration data is queried in real-time and not persisted beyond the tool call response
- Security event logs are retained in Application Insights per the configured retention period (default 90 days)
- Compliance artifacts committed to Git are retained indefinitely in the repository

## Output Data Policy (A002)

### Output Ownership

All outputs generated by the agent (compliance reports, gap analyses, POA&M entries, governance documents) are owned by the project owner (Pete).

### Output Sanitization

All agent outputs are processed through the `sanitize_output` function which redacts:
- Azure subscription IDs, tenant IDs, and object IDs
- Access keys, connection strings, and SAS tokens
- Private IP addresses and bearer tokens

### Output Deletion

Outputs can be deleted by:
- Removing compliance artifacts from the Git repository
- Clearing Application Insights logs
- Deleting Azure AI Foundry conversation history
