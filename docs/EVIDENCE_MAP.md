## Part 2: Type II Evidence Sampling Schedule

A SOC 2 Type II report covers a period of time (typically 6-12 months) and requires evidence that controls operated effectively throughout that period. This schedule defines how evidence will be collected over time.

### Sampling Frequencies

| Risk Level | Sampling Frequency | Examples |
| :--- | :--- | :--- |
| **High Risk** | Weekly | Agent output logs, security event logs, access control configurations |
| **Moderate Risk** | Monthly | Policy document versions, resource configuration snapshots, CI/CD run summaries |
| **Low Risk** | Quarterly | Vendor assessments, access reviews, tabletop exercise records |

### Evidence Collection Schedule

| Evidence Type | Frequency | Automated? | Collection Method | Retention |

| Agent compliance findings | Weekly | **Yes** | Azure Functions + App Insights | 365 days |
| Security event logs | Weekly | **Yes** | Azure Monitor / App Insights | 365 days |
| Resource configuration snapshots | Monthly | **Yes** | SOC 2 Learning Agent | 365 days |
| Policy document versions | Monthly | **Yes** | Git commit history | Indefinite |
| CI/CD run summaries | Monthly | **Yes** | GitHub Actions logs | 90 days |
| Access control review | Quarterly | No | Manual IAM review by Pete | 365 days |
| Vendor assessment review | Quarterly | No | Manual review by Pete | 365 days |
| Tabletop exercise records | Quarterly | No | Manual exercise by Pete | 365 days |

### Time-Bounded Export Requirements

For a SOC 2 Type II assessment, the following minimum retention periods apply:

-   **Audit logs**: 365 days minimum.
-   **Agent output logs**: 365 days minimum.
-   **Time-bounded evidence exports**: Last 90 days minimum for any given point-in-time assessment.
-   **Access review cadence**: Quarterly, with records retained for 365 days.
