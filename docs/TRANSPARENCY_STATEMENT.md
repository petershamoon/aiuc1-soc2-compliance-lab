# AI System Transparency Statement

## 1. System Purpose

The system consists of four autonomous AI agents designed to perform Governance, Risk, and Compliance (GRC) assessments of live Azure infrastructure. The agents evaluate Azure resource configurations against the SOC 2 Trust Services Criteria, governed by the 51 controls defined in the AIUC-1 standard.

This project serves as a portfolio piece to showcase the application of AI in a real-world compliance scenario.

## 3. Decision-Making Process

The system follows a **"tools provide data, agents provide judgment"** pattern.

1.  **Data Collection**: The Evidence Collector agent uses Azure Functions as tools to fetch raw, objective configuration data from Azure APIs (e.g., "Is public blob access enabled on storage account X?").
2.  **Analysis & Judgment**: The SOC 2 Auditor agent receives this raw data. It then uses its understanding of the SOC 2 criteria, as provided in its instructions, to make a compliance judgment (e.g., "Public blob access violates control CC5.").
3.  **Human-in-the-Loop**: For actions that modify the environment, such as the IaC Deployer agent applying a Terraform plan, a **mandatory human approval step** is enforced. The agent can generate and plan a change, but it cannot execute it without Pete's explicit approval.

## 4. Limitations & Known Issues

-   **No Real-Time Event Response**: The agents operate on a scheduled or on-demand basis. They do not monitor for and respond to Azure events in real-time.
-   **Potential for Hallucination**: While measures are in place (groundedness checks, known-answer tests), the LLMs may still hallucinate or generate factually incorrect information. All findings should be independently verified before being considered definitive.
-   **Scope-Bound**: The agents can only assess what they are instructed to look at. A misconfiguration in a resource not included in the Evidence Collector's target list will not be detected.
-   **No Self-Improvement**: The agents do not learn from experience or automatically update their own instructions. All improvements are manually implemented by Pete.
-   **Dependency on External APIs**: The system's functionality is entirely dependent on the availability and performance of the Azure AI Foundry and OpenAI APIs.
