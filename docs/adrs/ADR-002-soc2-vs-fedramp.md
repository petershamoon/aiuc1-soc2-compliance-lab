# ADR-002: Selection of SOC 2 as Initial Compliance Framework

**Status**: Accepted

**Context**: The AIUC-1 Compliance Lab project aims to demonstrate autonomous, AI-driven governance, risk, and compliance (GRC) assessment. A specific compliance framework is needed to serve as the basis for the assessment. The two primary candidates for a cloud-native, AI-driven project are SOC 2 and FedRAMP.

**Decision**: We will use **SOC 2** as the initial compliance framework for Version 1 of this project. This decision is made in accordance with AIUC-1 control E005.

**Alternatives Considered**:

1.  **FedRAMP**: The Federal Risk and Authorization Management Program, a U.S. government-wide program that provides a standardized approach to security assessment, authorization, and continuous monitoring for cloud products and services.
2.  **ISO 27001**: An international standard for information security management.
3.  **NIST Cybersecurity Framework (CSF)**: A set of voluntary standards, guidelines, and best practices to manage cybersecurity risk.

**Rationale**:

| Criteria | SOC 2 | FedRAMP | ISO 27001 | NIST CSF |
| :--- | :--- | :--- | :--- | :--- |
| **Relevance to AI/SaaS** | **Excellent**. Widely adopted by SaaS and technology companies, making it highly relevant to the project's narrative. | **Good**. Highly respected but specific to U.S. federal government suppliers. | **Good**. Broadly applicable but less focused on operational controls relevant to a SaaS environment. | **Good**. A foundational framework, but less of a formal, attestable standard and more of a set of guidelines. |
| **Flexibility** | **Excellent**. The Trust Services Criteria (TSC) are flexible and can be adapted to the specific risks of an AI-driven system. | **Fair**. The controls are prescriptive and extensive, which can be rigid for a v1 project. | **Good**. Flexible, but the ISMS requirement adds overhead. | **Excellent**. Very flexible, but this can make it difficult to define a clear assessment scope. |
| **Auditability** | **Excellent**. The attestation model (Type I and Type II reports) is well-understood and provides a clear structure for assessment. | **Excellent**. The authorization process is rigorous and well-defined. | **Excellent**. The certification process is globally recognized. | **Fair**. Not designed as a formal audit framework. |
| **Implementation Overhead** | **Medium**. The scope can be tailored to the system, making it manageable for a v1 project. | **High**. The control set is large and requires significant effort to implement and document. | **High**. Requires establishing a full Information Security Management System (ISMS). | **Low**. Can be adopted incrementally. |

**Conclusion**: SOC 2 provides the best balance of industry relevance, flexibility, and auditability for the initial version of this project. Its focus on operational controls and risk management aligns well with the goal of demonstrating AI-driven GRC. While FedRAMP is a robust framework, its high overhead and specific focus on federal suppliers make it less suitable for a v1 portfolio project. ISO 27001 and NIST CSF are valuable frameworks, but SOC 2's attestation model provides a clearer path to demonstrating compliance for a SaaS-like system.
